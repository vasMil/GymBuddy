Γενικά:

Για αυτά που έγραψα ίσως υπάρχουν και άλλα που δεν μου έρχονται αυτή τη στιγμή. Για τα υπόλοιπα που μένουν σκέψου κανένα με αυτό το κριτήριο και θα σου κάνω ένα review ακόμα.

Νομίζω το κειμενάκι που γράφεις για κάθε κλάση μπορεί να εμπλουτιστεί αρκετά.

Λείπουν πράγματα από το διάγραμμα, όπως αυτά που έχει στην εκφώνηση για το domain model, αριθμοί πάνω στα βελάκια που λένε πόσα αντικείμενα της μιας κλάσης αντιστοιχούν σε μια άλλη κλάση κλπ.

Δεν έχω διαβάσει καθόλου για aggregation και τα βελάκια, οπότε δεν μπορώ να κάνω κάποιο σχόλιο αυτή τη στιγμή. Θα τα διαβάσω και θα μιλήσουμε.

Επίσης έχει κάποια +,- στα αριστερά των attribute (πάλι στο παράδειγμα της εκφώνησης) τα οποία θυμάμαι από διάλεξη ότι δηλώνουν αν το attribute (ή το method) είναι public, private αντίστοιχα. Αν δεις τη διάλεξη έχει συμβολισμό και για τα protected (#) αλλά και για τα package (~). Αν θυμάμαι καλά το package είναι της java και δεν νομίζω να το χρειαστούμε, δες τι είναι για καλό και για κακό.

Στη διάλεξη επίσης αναφέρεις πράγματα για τον τύπο του κάθε attribute, άρα λογικά και εμείς θα πρέπει να βάλουμε κάτι τέτοιο στο domain model.

Γενικά το domain model είναι το στατικό μοντέλο το οποίο εξελίσσουμε μέσω της ICONIX τεχνικής. Είναι με άλλα λόγια ο λόγος για τον οποίο κάνουμε ότι κάνουμε στα use case και τα robustness. Πρέπει λοιπόν να περιέχει τα πάντα από όλα τα use case και όλα τα robustness, σε αρκετά μεγάλη λεπτομέρεια. Γνώμη μου λοιπόν είναι να ρίξεις μια ματιά στην έβδομη διάλεξη για να δεις τι έχει πει στη διάλεξη. Έχει αρκετά παραδείγματα και γενικά σου δίνει να καταλάβεις τι θέλει.

Αν διαφωνείς με κάποια διόρθωση ή βλέπεις ότι δεν προλαβαίνεις στείλε ένα μήνυμα στην ομαδική για να οργανωθούμε καλύτερα.
